## Affected Pages/Routes
- CLI: `python -m tests.visual_harness.g1_runner --suite representative ...`
  Verify the backend-aware G1 evidence path, resolved artifact root, and
  backend identity surfaced for comparison.
- CLI: `python -m tests.realman.visual_harness_runner --suite representative ...`
  Verify the collateral guard path, resolved artifact root, and publication
  policy for Realman representative evidence.
- Docs/planning: `70.1-VERIFICATION.md`, `docs/runtime.md`,
  `docs/grasp/sonic_paths.md`, `.planning/PROJECT.md`, `.planning/ROADMAP.md`,
  `.planning/STATE.md`
  Verify publication surfaces update only from the final chosen evidence root.

## Key Interactions to Verify
- G1 runner accepts or intentionally rejects `--control-backend` /
  `--sonic-path` with actionable guidance.
- G1 suite report exposes backend identity and provenance required by the
  comparator.
- Both runners print or write the resolved artifact root after timestamped
  output-root rewriting.
- Exploratory runs do not rewrite tracked baseline markdown.
- Final publication mode refreshes baselines/docs only from the chosen evidence
  root.

## Edge Cases
- Missing or empty suite output directory.
- Missing `suite_report_representative.json`.
- Missing or partial per-case `autonomous_report.json` files.
- Mismatched case count versus the expected 8 representative cases.
- Realman rerun introduces new execution errors or worsens accepted latency
  debt.
- `PROJECT.md` remains stale even after `ROADMAP.md` / `STATE.md` advance.

## Critical Paths
- Wave 1 backend-aware G1 harness contract lands before Wave 2 evidence reruns.
- G1 WBC run -> G1 SONIC compat run -> compare against Phase 70 baseline ->
  named decision in `70.1-VERIFICATION.md`.
- Realman collateral run -> compare against prior representative baseline ->
  block on new pass regression or execution error.
- Publication path -> final chosen evidence root -> baselines/docs/planning
  surfaces updated together.
