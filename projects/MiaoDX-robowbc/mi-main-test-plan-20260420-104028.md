## Key Interactions to Verify

- `scripts/bench_nvidia_official.sh --list-cases` enumerates the same case ids the docs publish.
- Official-wrapper execution emits normalized artifacts for every enabled case.
- Robowbc benchmark execution emits the same schema and preserves provider, commit, host, and warmup metadata.
- Decoupled benchmark fixtures cleanly split `walk_predict` and `balance_predict`.
- CLI end-to-end runs for `configs/sonic_g1.toml` and `configs/decoupled_g1.toml` produce stable JSON reports that can be normalized into the comparison artifact format.
- Published docs replace `TBD` placeholders with rows that point back to real artifact paths.

## Edge Cases

- Upstream model downloads drift because the helper scripts still fetch from upstream `main`.
- Official runtime does not expose a clean seam for the GEAR-Sonic encoder+decoder tracking path.
- Decoupled walk/balance command threshold does not map cleanly to the official implementation's mode switch.
- CLI reports omit fields required by the normalized comparison schema.
- Relative model paths work in CLI runs but fail in benchmark subprocesses.
- Official and robowbc artifacts disagree on units or percentile naming.
- A case id exists in docs but not in the case registry, or vice versa.
- A GPU/TensorRT row is published without matching provider parity on both stacks.

## Per-Case Requirements

- `gear_sonic_velocity/cold_start_tick`: benchmark result plus provenance, same provider on both stacks.
- `gear_sonic_velocity/warm_steady_state_tick`: benchmark result plus explicit note that planner stays idle.
- `gear_sonic_velocity/replan_tick`: benchmark result plus explicit note that planner executes.
- `gear_sonic_tracking/standing_placeholder_tick`: benchmark result plus semantic caveat if official seam is wrapper-only.
- `decoupled_wbc/walk_predict`: benchmark result with command fixture above switch threshold.
- `decoupled_wbc/balance_predict`: benchmark result with command fixture below switch threshold.
- `gear_sonic/end_to_end_cli_loop`: JSON report normalization, aggregate metrics, and raw artifact path.
- `decoupled_wbc/end_to_end_cli_loop`: JSON report normalization, aggregate metrics, and raw artifact path.

## Ship Criteria

- Every published comparison row is backed by a raw artifact.
- Every raw artifact includes commit pin, provider, and host metadata.
- Every doc table and status surface uses the same case ids and wording.
- Negative or mixed results map to a documented next action rather than ad hoc narrative spin.
