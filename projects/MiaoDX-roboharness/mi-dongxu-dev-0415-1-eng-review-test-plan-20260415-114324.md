## Edge Cases
- Missing baseline image for one selected view — report should show a labeled placeholder plus rerun guidance, not crash.
- Missing current capture for one selected view — report should show a labeled placeholder plus rerun guidance, not crash.
- Manifest/view mismatch — report should show an inline diagnostic banner and skip the broken pair.
- No failing phase / pass state — report should show explicit success copy and no empty comparison chrome.
- Still-image ambiguity — report should surface the ambiguity warning instead of overstating proof.
- Optional live-validation lane unavailable — test must be guarded with `pytest.importorskip("mujoco")` or bound to the MuJoCo-enabled CI lane so default `pytest -q` still passes in `[dev]`.

## Critical Paths
- Deterministic fixture lane: known-bad fixture -> evaluator fail -> manifest `approach` -> evidence resolver -> paired report cards.
- Happy pass lane: blessed baseline metrics + aligned visuals -> evaluator pass -> explicit success state in summary.
- Partial evidence lane: one selected image missing -> partial state -> placeholder + clear problem/cause/fix copy.
- Contract mismatch lane: selected phase/view missing or unsafe path -> mismatch banner -> no silent mispairing.
- Live MuJoCo lane: generated run under demo-enabled env reproduces the same machine contract as the deterministic fixture lane.
