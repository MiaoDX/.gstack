## Key interactions to verify

- **Sim server responds to all three endpoints** (observe, move, done) and captures
  frames to trace.jsonl with correct `seen_by_agent` flag.
- **Agent-consumption badges** visible in `report.html` — green border + 👁 for
  `/observe`, grey border + 🚶 for server-side `/move` snapshots.
- **Human interjection** — typing in terminal mid-run shows up in agent's next
  tool response and in the HTML tool-call log.
- **Workspace reset** — back-to-back runs against a long-lived gateway do NOT
  show memory contamination from the previous run.
- **Clean shutdown** — Ctrl+C terminates server, container, and daemon threads
  within 2 s.

## Edge cases

- `/move` called before any `/observe` → 200 with `server_warning`, NOT a 400.
- `/done` called twice → second call is a no-op 200, not an error.
- Human-message queue cap (10 entries) → oldest silently drop + `queue_overflow`
  event in trace.
- Agent enters infinite reasoning without calling any tool → wall-clock cap fires,
  `terminated_by="wall_clock"`, partial `final_message`.
- Gateway container killed mid-run → `terminated_by="error"`, clean client teardown.
- No TTY (stdin is a pipe) → stdin reader thread skips cleanly, logs
  "interjection disabled (no TTY)".

## Critical paths (must work end-to-end)

1. Kickoff → tool calls → done → replay.gif + report.html rendered
2. Kickoff → human interjection via stdin → agent acknowledges in next response
3. Back-to-back runs → second run starts with clean agent state
4. SIGINT during run → clean teardown, no zombie processes

## Regression guards (CRITICAL)

- **After `start_run()`, `step()`'s timeout MUST still default to 180 s.**
  The long wall-budget timeout is per-call, not a mutation of the shared
  httpx client. Without this guard, every existing push-mode caller
  inherits the 10-min timeout silently.
- **Push-mode paths unchanged**: `openclaw_demo.py`, `territory_game.py`,
  `coverage_game.py`, `bridge.step()` keep working identically. Run the
  existing test suite after every T52 change.

## Evals required

None — no prompt-template changes in this phase. The kickoff prompt is new
but there's no LLM-quality regression baseline yet; first run establishes one.
