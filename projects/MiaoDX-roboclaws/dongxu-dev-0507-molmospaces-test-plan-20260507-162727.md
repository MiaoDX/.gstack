# MolmoSpaces Manipulation Spike - Test Plan

Generated by `/autoplan` on 2026-05-07.

## Scope

This test plan covers the future MolmoSpaces manipulation spike described in
`docs/plans/molmospaces-manipulation-spike.md`. It is not an implementation log.

## Contract Tests

- MCP tool schemas for `observe`, `scene_objects`, `reach`/`goto`, `pick`,
  `place`, `open`, `close`, and `done` using a fake MolmoSpaces backend.
- Additive `trace.jsonl` fields remain compatible with existing trace fixture
  expectations.
- `run_result.json` records backend, scenario id, final status, artifact paths,
  and primitive provenance.

## Unit Tests

- Scenario JSON parsing and seeded determinism.
- Private scoring manifest parsing.
- State-delta scorer for restored, missed, invalid, lost, and timeout cases.
- Object/receptacle ID stale-reference handling.
- Report context generation for before/after images, restored/missed table,
  trace summary, and provenance disclosure.

## Local-Dev Validation

These require a local workstation with MolmoSpaces/MuJoCo assets available:

- Capability spike imports/runs the smallest MolmoSpaces example.
- Scene loading and camera frame capture work.
- Object inventory and object state readback work.
- Pick/place primitive works, or scripted/shim primitive is explicitly labeled.
- Layer 1 coding-agent run produces `trace.jsonl`, `run_result.json`, and
  `report.html`.
- Layer 2 cleanup run restores at least 3 of 5 misplaced objects in an easy seed.

## Non-Goals

- Multi-agent MolmoSpaces testing.
- Territory/coverage on MolmoSpaces.
- Low-level arm or VLA action expert validation.
